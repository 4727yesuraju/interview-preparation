# Software Engineering Notes

## Microservices

### Definition

**Microservices** is an architectural approach where a large application is split into multiple **small, independent services**, with each service responsible for **one specific business function**.

### Key Points

- Each service has a single responsibility.
- Services are developed and deployed independently.
- Services communicate using APIs or message queues.
- Each service can be scaled separately.

---

## Non-blocking I/O

### Definition

**Non-blocking I/O** is a programming model where a program starts an input/output (I/O) operation and **continues executing other tasks instead of waiting** for the operation to finish.

### Key Points

- Does not block the program while waiting for I/O.
- Improves application performance and responsiveness.
- Commonly used for file operations, database queries, and network requests.
- Enables efficient handling of many concurrent requests (e.g., in Node.js).

---

# libuv

## Definition

**libuv** is a **C library** used by **Node.js** to handle **asynchronous (non-blocking) I/O operations** such as file system access, networking, timers, and thread pool tasks.

---

## Responsibilities

libuv is responsible for:

- 📂 File System Operations (`fs.readFile`, `fs.writeFile`, etc.)
- 🌐 Network I/O (HTTP, TCP, UDP)
- ⏲️ Timers (`setTimeout`, `setInterval`)
- 🧵 Thread Pool Management
- 🔄 Event Loop Management

---

## How it Works

```text
JavaScript Code
       │
       ▼
Node.js Runtime
       │
       ▼
     libuv
       │
       ▼
File System / Database / Network
       │
       ▼
Callback returned to Event Loop
```

---

## Example

```javascript
const fs = require("fs");

console.log("Start");

fs.readFile("data.txt", (err, data) => {
  console.log("File Read");
});

console.log("End");
```

### Execution Flow

1. `console.log("Start")` executes.
2. `fs.readFile()` is called.
3. Node.js sends the file read request to **libuv**.
4. **libuv** performs the file operation asynchronously.
5. JavaScript continues executing and prints `"End"`.
6. Once the file is read, **libuv** places the callback into the **event loop**.
7. The event loop executes the callback and prints `"File Read"`.

### Output

```text
Start
End
File Read
```

---

## Why libuv?

Without **libuv**, Node.js would have to wait for every file read, database query, or network request to finish before executing the next line of code.

With **libuv**:

- JavaScript never waits for I/O operations.
- The CPU can execute other tasks while I/O is in progress.
- Node.js can efficiently handle thousands of concurrent requests.

---

## Interview Definition

> **libuv is a C library that powers Node.js's event loop and enables asynchronous, non-blocking I/O by handling file system operations, networking, timers, and thread pool tasks in the background.**
